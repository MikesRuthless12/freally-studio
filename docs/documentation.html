1. Getting Started / Install

Download and launch Freally Studio in under two minutes.

  1. Download the installer for your OS from the Releases page: Windows (.exe), macOS Apple Silicon (.dmg), macOS Intel (.dmg), or Linux (.AppImage / .deb).
  2. Run the installer. On Windows, double-click the .exe and follow the prompts. On macOS, open the .dmg, drag Freally Studio to your Applications folder, then double-click it. On Linux, make the AppImage executable (chmod +x Freally*.AppImage) and run it, or install the .deb with sudo dpkg -i.
  3. Allow audio access when prompted. Freally Studio needs microphone/audio device access to play and record sound.
  4. Click anywhere on the app when it first opens. This is required to resume the Web Audio context in your browser or Electron shell.
  5. Verify playback works by pressing Space on an empty project — the transport should start (no sound yet, but no errors in the title bar).
Browser build: If you're running from source, install dependencies with pnpm install (or npm install), then start the dev server with pnpm dev and open http://localhost:5173 in Chrome or Edge. Firefox is supported but Chrome gives the best Web Audio performance.

2. Create Your First Project

Set up a new session and pick your musical key, scale, and tempo.

  1. Open Freally Studio. A blank project loads automatically — no file dialog needed.
  2. Set your Genre using the Genre selector in the top control bar. There are 35+ options spanning Trap, Hip Hop, House, Techno, Drum & Bass, Jazz, and more. Your choice shapes the patterns that Freally Studio generates.
  3. Set your Key (C through B, with sharps) and Scale (Major, Minor, Dorian, Pentatonic Minor, Blues, etc.). All generators stay in key automatically.
  4. Set your Bars — choose 1, 2, 4, or 8 bars for the loop length of each generated pattern.
  5. Pick your Grid Resolution: 1/16 for standard patterns, 1/32 for finer rhythmic detail (uses slightly more CPU).
  6. You're ready. Projects save state automatically in your browser's IndexedDB storage, so your samples and arrangements persist between sessions without manual saving.

3. Generate a Chord Progression

Create a professional chord progression matched to your genre, key, and scale.

  1. Click the Chords tab in the MIDI Generator panel.
  2. Load an instrument sample (optional but recommended): click the instrument panel in the Chords tab, browse to an audio file (WAV, MP3, OGG, or FLAC), and select it. Alternatively, drag a file from the File Explorer panel directly onto the instrument panel.
  3. Confirm your Genre, Key, Scale, and Bars in the top control bar.
  4. Click GENERATE. Freally Studio builds a chord progression that fits the genre and scale. The chord grid populates with color-coded blocks showing chord names, durations, and positions.
  5. Preview the progression by pressing Space in the generator view to hear it loop.
  6. Edit individual chords in the grid if needed — click a chord block to adjust its root, inversion, or duration.
  7. Drag the pattern to the Arrangement view once you're happy — grab the generated clip and drop it onto a chord track in the timeline.

4. Generate a Melody / Bassline

Generate a scale-aware melody or a bassline that follows your chord progression.

  1. Click the Melody tab in the MIDI Generator panel.
  2. Load an instrument sample for melody playback by clicking the instrument panel or dragging a sample file onto it (WAV, MP3, OGG, FLAC supported).
  3. Confirm your Genre, Key, Scale, Bars, and Octave. The Octave control (C0–C8) lets you target the right register for a lead melody versus a bassline.
  4. Click GENERATE. Freally Studio produces a melodic line or bassline using scale-aware note selection and genre-specific phrasing. The note grid populates with individual note blocks.
  5. Preview and edit — press Space to hear it loop. Click individual notes in the grid to move, resize, or delete them.
  6. For a bassline: set the Octave to C1 or C2, choose a bass-heavy sample (e.g., an 808 or sub bass), and generate. The melody generator produces bassline-appropriate patterns when the register is low.
  7. Drag the pattern to the Arrangement view onto a melody or bass track in the timeline.

5. Generate Drums / a Pattern

Build a complete drum pattern with 8 elements across 35+ genre presets.

  1. Click the Drums tab in the MIDI Generator panel.
  2. Load samples for each drum element (optional):
    1. Click a drum element label (Kick, Snare, Clap, Hi-Hat Open, Hi-Hat Closed, Tom, Crash, or Ride).
    2. Browse to an audio file and select it, or drag a sample file from the File Explorer panel directly onto the drum slot.
    3. A waveform preview appears in the slot confirming the sample is loaded.
  3. Set Genre, Key, Scale, and Bars in the top control bar. The genre especially drives which rhythmic patterns are used (e.g., Trap produces rolling hi-hats; House produces four-on-the-floor kicks).
  4. Click GENERATE. A 16-step (or 32-step) grid fills with hits for each drum element according to genre-specific professional patterns, variations, and fills.
  5. Edit the pattern by clicking grid cells to toggle individual hits on or off.
  6. Preview by pressing Space. The pattern loops so you can audition edits in real time.
  7. Drag the drum clip to the Arrangement view onto a drum track in the timeline.

6. Use the Sample Library

Import, browse, tag, and preview your audio files in the built-in File Explorer.

  1. Open the File Explorer panel by clicking the folder icon in the left sidebar. It shows a collapsible folder tree of your imported sample folders.
  2. Import a folder: click the + button at the top of the File Explorer. A system folder picker opens. Select a folder containing your audio files (WAV, MP3, OGG, FLAC). Freally Studio indexes it and shows all files in the tree.
  3. Navigate the tree — click folder names to expand or collapse them. File count badges show how many audio files are in each folder.
  4. Preview a sample — click any audio file in the tree to hear it play back immediately. A stop button appears while it plays. A waveform preview renders in the panel.
  5. Use the search bar to filter by filename across all imported folders.
  6. Drag a sample to a generator — drag any file from the Explorer directly onto a drum slot, the Chords instrument panel, or the Melody instrument panel to load it.
  7. Analyze a folder for auto-generation: right-click a folder in the tree and choose Analyze & Generate. Freally Studio detects the key, BPM, genre, and scale from the files inside and pre-sets the top control bar. Then click GENERATE to create a matching pattern.
  8. Samples persist across sessions — IndexedDB caches your imports, so you don't need to re-import after restarting the app.
Best results with folder analysis: use folders with 5 or more files of a single style. Name folders with genre keywords (e.g., "Trap Dark" or "Lo-Fi Chill") to help the analyzer. Avoid mixing wildly different genres in one folder.

7. Edit on the Arrangement View

Arrange clips on a multi-track timeline, set loop regions, and build your full song structure.

  1. Switch to Arrangement view by clicking the Arrangement icon (timeline icon) in the main navigation.
  2. Drag clips from the generator panels onto the timeline tracks. Clips show sample names above them and waveform previews inside.
  3. Reposition clips by dragging them left or right on the timeline.
  4. Set a loop region: drag the loop markers at the top of the timeline to define the start and end of your loop. Press Ctrl+L to toggle looping on or off.
  5. Press Space to play or stop. Press Ctrl+Space to jump back to the start and play from the beginning.
  6. Right-click a clip for context options: duplicate, delete, rename, or open in the clip editor.
  7. Duplicate a clip by selecting it and pressing Ctrl+D.
  8. Delete a selected clip with the Delete key.
  9. Use keyboard shortcuts to set loop boundaries quickly: press [ to mark the current playhead position as the loop start, and ] to mark the loop end.

8. Mixing & Mastering

Balance levels, pan instruments, add effects, and polish the master bus.

  1. Switch to Mixer view by clicking the Mixer icon in the main navigation. The FL Studio-style vertical channel strip layout appears.
  2. Adjust channel volume by dragging the vertical fader on each strip. VU meters show real-time peak levels.
  3. Pan a channel using the pan knob at the top of each strip. Spread drums wide, keep kick and bass centered for a full stereo image.
  4. Solo or mute a channel with the S (Solo) and M (Mute) buttons. LED indicators show which channels are active. Soloing a channel lets you hear it in isolation.
  5. Add effects to a channel: click the + button in the channel's plugin rack. A plugin menu appears — choose from the available effects (EQ, compressor, reverb, delay, and more).
  6. Adjust effect parameters in the plugin panel that opens below the rack slot. Drag knobs or type values to dial in your sound.
  7. Apply the 3-band EQ directly to any clip via the Audio Clip Editor — right-click a clip in the Arrangement and choose Edit to access trim, fade, normalize, reverse, EQ, and filter controls.
  8. Process the master channel using the dedicated Master strip on the far right of the mixer. Apply a limiter or master EQ here to control final output level and prevent clipping.
  9. Color-code channels by right-clicking a channel header and selecting a color — this helps you visually organize drums, bass, melody, and effects busses.

9. Render & Export Your Track

Export your beat or individual stems as WAV, MP3, or MIDI.

  1. Set your loop region in the Arrangement view to define what gets exported. Make sure the loop markers bracket the section you want.
  2. Open the Export dialog by clicking the Export button in the top toolbar (or via File → Export).
  3. Choose your export format:
    1. WAV — high-quality, lossless audio. Best for dropping into a DAW.
    2. MP3 — compressed audio for sharing online.
    3. MIDI — exports the MIDI pattern from the active generator for use in any DAW or instrument.
  4. Select stem export or full mix. Exporting individual stems gives you one file per track (drums, chords, melody, bass), which is useful for post-processing in an external DAW.
  5. Click Export / Download. The file downloads to your default Downloads folder.
Tip: Export in WAV at 44100 Hz / 24-bit for the best quality when you plan to master the track further in another DAW. Keep individual sample files under 10 MB for smooth real-time playback during the session.

10. Collaborate

Work on a session with another person in real time over a P2P connection.

  1. Open the Collaborate panel from the top navigation or toolbar.
  2. Create a room by clicking New Session. Freally Studio generates a secure room link and a 6-digit PIN.
  3. Share the room link and PIN with your collaborator. The PIN is delivered as a URL fragment and never leaves the browser — the server cannot see it.
  4. Your collaborator opens the link in their browser and enters the PIN when prompted. Once authenticated, they join the session.
  5. Both users can now edit the arrangement, tweak patterns, and adjust settings. Changes sync peer-to-peer in real time.
  6. Mobile link: if you want to join from a phone, open the room link on the mobile browser. The same 6-digit PIN is required. Note that mobile is a companion / controller view — full production is best on desktop.
  7. End the session by closing the Collaborate panel or navigating away. The room expires automatically after the last participant leaves.
Note: Freally Studio is completely free — every feature, including real-time collaboration, for everyone. No Pro tier, no subscriptions, no payments, no ads.

11. Settings

Customize Freally Studio to match your workflow and hardware.

  1. Toggle Dark / Light theme by clicking the sun/moon button in the app header. All panels and generators adapt automatically. Your preference is saved across sessions.
  2. Change audio output device in Settings → Audio. Select your interface, headphones, or speakers from the dropdown. Freally Studio uses the Web Audio API and respects your OS default unless you override it here.
  3. Set sample rate in Settings → Audio. The recommended values are 44100 Hz or 48000 Hz. Mismatched sample rates can cause pitch drift.
  4. View app version and build info in Settings → About. The project creation date (February 15th, 2026) is recorded here alongside the current version number.
  5. Clear sample cache in Settings → Storage if the app becomes slow or you want to free disk space. Your project arrangement is not affected — only cached audio files are removed (you will need to re-import samples).

12. Tips & Troubleshooting

Common issues and performance tips to keep your sessions running smoothly.

  1. No audio playback: click anywhere on the app page to resume the Web Audio context (browsers require a user gesture before audio can start). Also verify your OS volume is not muted and that the correct output device is selected in Settings → Audio.
  2. Sample won't load: confirm the file is an audio format (WAV, MP3, OGG, or FLAC). Keep individual files under 10 MB. If a file fails silently, try converting it to WAV first.
  3. Folder analysis gives wrong key or genre: the detector works best with 5+ files of a consistent style. Try right-clicking a more specific sub-folder. You can also override the detected values manually in the top control bar before generating.
  4. App is slow or stuttering during playback: switch the Grid Resolution to 1/16 (instead of 1/32), limit active tracks to 8–16, close unused browser tabs, and use WAV files (they decode faster than MP3). If running from source, ensure you are using the production build (pnpm build) rather than the dev server for best performance.
  5. Dev server port already in use: run pnpm dev --port 3000 (or any other free port) to start on a different port.
  6. Dependencies failed to install: delete node_modules and pnpm-lock.yaml, then run pnpm install again to get a clean install.
  7. Generated patterns don't match your style: try a more specific genre, adjust the key and scale to match your reference track, or use Folder Analysis on a folder of samples from the style you want. Then regenerate.
  8. Performance best practices:
    • Use WAV files for best decode performance.
    • Keep samples under 10 MB each.
    • Use 1/16 grid resolution over 1/32 when high note density is not needed.
    • Limit your project to 8–16 active tracks for smooth real-time playback.
    • Close the File Explorer panel when not in use to free rendering resources.
