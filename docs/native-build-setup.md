# Native addon build setup (TASK-D01)

Freally Studio ships two cmake-js native addons:

| Addon | Path | Output | Notes |
|---|---|---|---|
| VST3 host | `native/` | `native/build/Release/freally_vst3_host.node` | needs the `native/third_party/vst3sdk` git submodule (recursive) |
| Audio capture | `native/audio_capture/` | `native/audio_capture/build/Release/freally_audio_capture.node` | WASAPI code is `_WIN32`-guarded; compiles as a stub on macOS/Linux |

## Known-good local toolchain (verified 2026-07-11, Windows 11)

| Component | Version | Where |
|---|---|---|
| Visual Studio | Community 2026 (18.7.11925.98) with the **Desktop development with C++** workload | `C:\Program Files\Microsoft Visual Studio\18\Community` |
| MSBuild | 18.7.8 | ships with VS |
| Windows SDK | 10.0.26100.0 | VS installer component |
| CMake | 4.4.0-rc2 (any ≥ 3.15 works; the CMakeLists set `cmake_minimum_required(3.15)`) | on `PATH` |
| Node | 26.5.0 | system |
| cmake-js | 7.4.0 (declared `^7.3.0`) | `native/node_modules` |
| node-addon-api | 8.6.0 (declared `^8.0.0`) | `native/node_modules` |

macOS: Xcode command-line tools + CMake (Homebrew). Linux: `build-essential cmake`.

## Build commands (from the repo root)

```bash
npm --prefix native install        # once, or after dependency changes
npm run native:rebuild             # VST3 host addon
npm run audio-capture:rebuild      # audio capture addon
```

`native/build-addon.js` auto-detects a Visual Studio-bundled CMake when none
is on `PATH` (checks VS `18`/`2022`/`2019`/`2017`, all editions).

## The `Unknown CMake command CMAKE_DETERMINE_COMPILER_ID` failure — root cause

The failure recorded as CHANGELOG deferred item #1 was caused by a **stale
`build/` directory**: a `CMakeCache.txt`/`CMakeFiles/` tree generated by one
CMake/VS pairing being re-configured by a different one. CMake then resolves
internal modules from the mismatched cache and dies with errors like
`Unknown CMake command CMAKE_DETERMINE_COMPILER_ID`.

**Fix (always safe):** delete both build trees and rebuild.

```powershell
Remove-Item -Recurse -Force native\build, native\audio_capture\build
npm run native:rebuild
npm run audio-capture:rebuild
```

Verified 2026-07-11: both addons compile from a clean tree with the toolchain
above, the VST3 host's 43 self-tests pass, and the A7 bounds-check patch in
`native/src/napi_plugin_wrapper.cpp` is compiled into the binary.
`.github/workflows/release.yml` already wipes both build trees before every
CI build for the same reason ("never reuse a committed/stale CMake cache") —
`ci.yml`'s `native` job does the same.

## CI

`.github/workflows/ci.yml` job `native` builds BOTH addons on
windows-latest / macos-latest / ubuntu-latest on every PR (and pushes to
master) and uploads the `.node` binaries as artifacts. Checkout must use
`submodules: recursive` for the vst3sdk.
